# 项目架构分析：nanobot 框架与多平台智能客服 Bot

## 一、整体层次结构

```
D:\Agent_Project\nanobot-main\nanobot-main\
│
├── nanobot/              ← 【公用层】nanobot 框架源码（不动）
├── workspace/            ← 【项目专属层】你的智能客服 Bot（你写的）
├── venv/                 ← Python 虚拟环境（框架 + 依赖）
├── docs/                 ← 框架文档
├── tests/                ← 框架测试
└── pyproject.toml        ← 框架包定义
```

## 二、公用部分 vs 非公用部分

### 公用层：`nanobot/` 框架核心（pip install 后也可位于 venv/Lib/site-packages/nanobot/）

这是 nanobot 框架本身，**你无需修改，升级框架时直接替换**。

```
nanobot/
├── agent/                # 核心 Agent 引擎
│   ├── loop.py           #   主循环：用户消息 → LLM → 工具调用 → 回复
│   ├── context.py        #   上下文构建器：拼装 system prompt
│   ├── memory.py         #   MemoryStore(读写 MEMORY.md)、Consolidator(压缩历史)、Dream(两阶段记忆处理)
│   ├── skills.py         #   技能加载器：发现 workspace/skills/*/SKILL.md
│   ├── runner.py         #   LLM 调用运行器
│   └── tools/            #   内置工具集（grep, glob, read_file, write_file, exec 等）
│       ├── filesystem.py
│       ├── registry.py
│       ├── shell.py
│       ├── mcp.py         #   MCP 客户端（连接你的 ticket_server.py）
│       └── sandbox.py
│
├── channels/             # 16 个内置通道实现（你只改了 config 启用/禁用）
│   ├── base.py           #   通道基类：Inbound → Agent → Outbound
│   ├── manager.py        #   ChannelManager：发现、启动、重试、消息分发
│   ├── registry.py       #   通道注册表：扫描所有通道
│   ├── feishu.py         #   飞书（lark-oapi SDK + WebSocket 长连接）
│   ├── dingtalk.py       #   钉钉（dingtalk-stream SDK + Stream Mode）
│   ├── telegram.py       #   Telegram（python-telegram-bot）
│   └── ...               #   其他 13 个通道
│
├── providers/            # LLM 供应商适配（DeepSeek、OpenAI、Anthropic 等）
├── bus/                  # 消息总线（InboundMessage / OutboundMessage + asyncio.Queue）
├── cli/                  # CLI 入口（nanobot gateway、nanobot agent、nanobot onboard）
├── config/               # 配置加载 + schema 校验
├── session/              # 会话管理（跨通道统一会话）
├── templates/            # system prompt 模板（identity、dream、memory 等）
├── cron/                 # 定时任务（Dream、Heartbeat）
├── security/             # 沙箱 + 访问控制（allowFrom 白名单）
└── web/                  # Web UI（非 gateway 模式）
```

**你的交互方式**：只通过 `config.full.json` 控制通道的启用/禁用和凭证，从不修改 `nanobot/` 下的代码。

### 项目专属层：`workspace/`（你的智能客服 Bot）

这是你实际开发的部分，通过 `--config` 和 `--workspace` 参数指定。

```
workspace/
│
├── config.full.json          # 【配置】多通道完整配置（飞书+钉钉）
├── config.json               # 【配置】本地测试配置（纯 CLI）
│
├── AGENTS.md                 # 【身份】Agent 身份定义 + 行为准则 + 工作流程
│   ├── Identity              #   你是 CloudSync 产品客服专家
│   ├── Core Principles       #   知识库优先 > 坦诚不知 > 记住用户 > 多语言 > 及时升级
│   ├── Workflow              #   读取记忆 → 分类问题 → 搜索/建单 → 回复 → 更新记忆
│   └── Boundaries            #   能力边界（不处理支付、不访问外网等）
│
├── SOUL.md                   # 【人格】Bot 的语气和行为风格（框架默认模板）
├── USER.md                   # 【用户档案】框架默认文件，本项目中未重度使用
├── TOOLS.md                  # 【工具说明】框架内置工具的用法提示
├── HEARTBEAT.md              # 【心跳】框架自动维护
│
├── skills/                   # 【技能】always: true 自动加载到 Agent 上下文
│   ├── faq/SKILL.md          #   FAQ 知识库查询 → grep knowledge_base/
│   ├── ticket/SKILL.md       #   工单管理 → MCP ticket server 的 4 个工具
│   └── memory/SKILL.md       #   记忆记录 → 识别关键信息自动写 MEMORY.md
│
├── knowledge_base/           # 【知识库】Markdown FAQ 文件，Agent 用 grep 搜索
│   ├── general.md            #   账号、注册、套餐、联系
│   ├── billing.md            #   支付、发票、退款、定价
│   ├── api.md                #   API 认证、限流、Endpoint、SDK
│   └── troubleshooting.md    #   常见报错、性能、移动端问题
│
├── mcp_servers/              # 【MCP Server】外部工具，通过 JSON-RPC over stdio 连接
│   └── ticket_server.py      #   工单系统（SQLite 存储，4 个工具）
│
├── memory/                   # 【记忆】框架自动维护
│   ├── MEMORY.md             #   长期记忆（多用户隔离，channel:chat_id 格式）
│   └── history.jsonl         #   对话历史（Consolidator 触发后写入）
│
└── cron/                     # 【定时任务】框架自动维护
    └── jobs.json             #   Dream(每2h)、Heartbeat(每30min)
```

## 三、与默认 workspace（~/.nanobot/workspace/）的区别

| 维度 | 默认 workspace | 你的项目 workspace |
|------|---------------|-------------------|
| 位置 | `~/.nanobot/workspace/` | `workspace/`（项目目录下） |
| 用途 | 个人通用助手 | 多平台智能客服 Bot |
| AGENTS.md | 默认模板（空壳） | **完整客服身份定义 + 工作流程** |
| skills/ | 框架内置技能（weather 等） | **faq + ticket + memory 三个业务技能** |
| knowledge_base/ | 无 | **4 个 FAQ 文件，280 行知识** |
| mcp_servers/ | 无 | **ticket_server.py（SQLite 工单系统）** |
| memory/ | 个人助手记忆 | **多用户隔离记忆** |
| config | 指向 `~/.nanobot/config.json` | 项目级 `config.full.json`（多通道凭证） |
| Git 追踪 | 框架默认 Git 追踪 | 项目 Git 提交 |

**本质差异**：默认 workspace 是框架给你参考的模板，你的 workspace 是填入了**业务规则、领域知识、自定义工具**的完整产品。

## 四、启动命令解读

```bash
nanobot gateway \
  --config workspace/config.full.json \   # 指定项目配置（供应商 + 三通道凭证）
  --workspace workspace                    # 指定项目 workspace（Agent 身份 + 知识库 + 技能）
```

- `nanobot` 命令 → 调用 `venv` 中安装的 nanobot CLI
- `gateway` → 启动 ChannelManager + AgentLoop + WebSocket 通道
- `--config` → 告诉框架用哪个 JSON 配置（不指定则用 `~/.nanobot/config.json`）
- `--workspace` → 告诉框架去哪里找 AGENTS.md、skills、knowledge_base

## 五、数据流（一次用户对话）

```
用户在飞书发 "如何重置密码"
        │
        ▼
feishu.py (WebSocket 长连接)
    ├── 解析飞书消息格式 → InboundMessage
    ├── 检查 allowFrom 白名单
    └── 投递到 MessageBus
            │
            ▼
ChannelManager._dispatch_outbound → AgentLoop
    │
    ├── ContextBuilder.build_system_prompt()
    │   ├── identity.md（运行时信息）
    │   ├── AGENTS.md（客服身份 + 工作流）
    │   ├── MEMORY.md（用户历史）
    │   ├── memory/SKILL.md（always:true）
    │   ├── faq/SKILL.md（always:true）
    │   ├── ticket/SKILL.md（always:true）
    │   └── Recent History（最近 50 条）
    │
    ├── LLM 调用（DeepSeek）
    │   ├── 判断：FAQ 可回答 → grep knowledge_base/
    │   ├── 召回：general.md → 密码重置步骤
    │   └── 回复给用户
    │
    └── memory skill 检查
        ├── 无关键信息 → 不写 MEMORY.md
        └── 有关键信息 → write_file("memory/MEMORY.md")
            │
            ▼
OutboundMessage → MessageBus → feishu.py → 飞书格式转换 → 用户看到回复
```

## 六、关键设计决策回顾

| 决策 | 原因 |
|------|------|
| 不使用 Flask Webhook | nanobot ChannelManager 原生处理多平台路由，WebSocket 无需公网 IP |
| MCP Server 用原始 JSON-RPC | 避免 MCP SDK Server 类版本兼容问题 |
| Skills 使用 `always: true` | 客服核心能力，每次对话必须加载 |
| 知识库用 Markdown + grep | FAQ 条目百条以内，无需向量数据库 |
| SQLite 做工单存储 | 零配置、文件级存储，备份简单 |
| 记忆用 memory skill 而非 Dream | DeepSeek 短对话不触发 Consolidator，Dream 管道空转 |
| 多用户用 `channel:chat_id` 隔离 | 钉钉 staff_id ≠ 飞书 open_id，跨平台不混淆 |
| 部分匹配先建单再追问 | 客服场景问题不能丢，建单是兜底安全网 |
