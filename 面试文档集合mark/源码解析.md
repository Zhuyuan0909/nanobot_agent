## 一、整体分层架构

```
┌────────────────────────────────────────────────────┐
│                   Channel 层                        │

│                 AgentLoop (agent/loop.py)           │

                       │
         ┌─────────────┼─────────────┐

┌──────────┐  ┌──────────┐  ┌──────────────┐
```

## 三、一次完整对话调用的数据流

```
用户输入 "你好"
  │
  ▼
AgentLoop.run()                           [loop.py:454]
  │ 从 MessageBus 拉取 InboundMessage
  ▼
AgentLoop._dispatch()                     [loop.py:521]
  │ 获取 session、加并发锁
  ▼
AgentLoop._process_message()              [loop.py:655]
  │
  ├─ session.get_history(max_messages=0)  → 全部未压缩消息
  ├─ auto_compact.prepare_session()       → 检查是否空闲会话（需恢复摘要）
  ├─ consolidator.maybe_consolidate()     → token 压力检测（超 50% 触发）
  │
  ▼
ContextBuilder.build_messages()           [context.py:129]
  │ 组装: identity + bootstrap(USER/SOUL/AGENTS) + MEMORY.md
  │       + always skills + skills summary + recent history + user_msg
  ▼
AgentLoop._run_agent_loop()               [loop.py:354]
  ▼
AgentRunner.run()                         [runner.py:227]
  │
  │  ┌─ for iteration in range(max_iterations):
  │  │
  │  ├─ 1. 上下文治理:
  │  │     _drop_orphan_tool_results()
  │  │     _backfill_missing_tool_results()
  │  │     _microcompact()                → 压缩重复工具结果
  │  │     _apply_tool_result_budget()    → 截断过长结果
  │  │     _snip_history()                → token 超限时裁剪历史
  │  │
  │  ├─ 2. _request_model()              → LLMProvider.chat_with_retry()
  │  │     发送 [system + history + user_msg] + tools 定义
  │  │
  │  ├─ 3. 收到 LLMResponse
  │  │     ├─ should_execute_tools?
  │  │     │   → _execute_tools() → 并发执行所有 Tool Call
  │  │     │   → 结果追加到 messages
  │  │     │   → continue 下一轮迭代
  │  │     │
  │  │     └─ 无 tool_calls? → 这是最终回复
  │  │         → 空回复重试（最多 2 次）
  │  │         → length 截断恢复（最多 3 次）
  │  │         → 检查 mid-turn injection
  │  │         → break
  │  │
  │  └─ 4. 异常: max_iterations / error / tool_error
  │
  ▼
返回 AgentRunResult (final_content, tools_used, messages, stop_reason)
  │
  ▼
_save_turn(session, all_msgs, ...)        → 工具结果持久化
session.save()                            → 写回 JSONL 文件
```

### 4.4 LLM Provider 层

#### base.py — 抽象基类
```python
class LLMProvider(ABC):
    @abstractmethod
    async def chat_with_retry(messages, tools, tool_choice) -> LLMResponse:
        """统一接口，内置重试策略"""
```

**LLMResponse 关键字段：**
```python
@dataclass
class LLMResponse:
    content: str | None
    tool_calls: list[ToolCallRequest]
    finish_reason: str          # "stop" | "tool_calls" | "length" | "error"
    usage: dict[str, int]       # prompt/completion/cached tokens
    reasoning_content: str | None   # DeepSeek-R1 / Kimi 思考内容
    thinking_blocks: list | None    # Anthropic extended thinking
    error_status_code / error_kind / error_type  # 错误元数据
```

**重试策略**（[base.py:94-120]()）：
- 瞬态错误识别：429、5xx、timeout、connection、overloaded
- 指数退避：(1, 2, 4) 秒
- 持久错误延迟上限 60 秒，相同错误最多重试 10 次
- 非重试错误：insufficient_quota、billing 相关

#### openai_compat_provider.py — 主力后端
兼容所有 OpenAI-API 格式的服务（DeepSeek、OpenRouter、Groq 等），通过 [AsyncOpenAI](https://github.com/openai/openai-python) SDK 通信。

### 4.5 工具系统

#### Tool 基类 — [agent/tools/base.py]()

```python
class Tool(ABC):
```

**关键设计：参数自动校验 + 错误回传给 LLM**
```python
# registry.py
async def execute(self, name, params):
    tool, params, error = self.prepare_call(name, params)
    if error:
        return error + "\n\n[Analyze the error above and try a different approach.]"
    result = await tool.execute(**params)
    if isinstance(result, str) and result.startswith("Error"):
        return result + "\n\n[Analyze the error above and try a different approach.]"
    return result
```
工具执行失败时返回错误信息 + hint，让 LLM 能够自我修正。

### 4.6 工具执行引擎 — [agent/runner.py]()

**AgentRunner.run()** 是 LLM ↔ Tool 循环的核心：

```python
async def run(self, spec: AgentRunSpec) -> AgentRunResult:
    messages = list(spec.initial_messages)
    for iteration in range(spec.max_iterations):
        # 1. 上下文治理（裁剪、压缩、修复）
        messages_for_model = self._govern_context(messages)

        # 2. 调用 LLM
        response = await self._request_model(spec, messages_for_model, hook)

        # 3. 判断响应类型
        if response.should_execute_tools:
            # 执行工具 → 结果追加到 messages → continue
            results = await self._execute_tools(spec, response.tool_calls)
            messages.extend(tool_result_messages)
            continue

        # 4. 空回复重试 / length 截断恢复
        if is_blank(clean):
            retry up to 2 times
        if finish_reason == "length":
            append "continue" message, retry up to 3 times

        # 5. 最终回复
        break
    return AgentRunResult(...)
```

**上下文治理**（每次 LLM 调用前执行）：
1. `_drop_orphan_tool_results()` — 移除孤立的 tool result
2. `_backfill_missing_tool_results()` — 补填丢失的 tool result
3. `_microcompact()` — 将重复的工具调用结果折叠
4. `_apply_tool_result_budget()` — 对过长工具结果截断
5. `_snip_history()` — token 超限时从历史头部裁剪

**并发工具执行：**
```python
# 多个 tool call 并行执行
results = await asyncio.gather(*[
    tool.execute(**tc.arguments) for tc in tool_calls
])
```

### 4.7 记忆系统 — [agent/memory.py]()

记忆系统分为三层，由三个组件协作完成：

```
短期记忆（完整对话）
  Session JSONL → session.get_history() → 直接注入 LLM 上下文
       │
       │ (token 压力 > 50% 时触发)
       ▼
中期记忆（对话摘要）
  Consolidator.archive() → memory/history.jsonl
       │
       │ (Dream 每 2h 定时触发)
       ▼
长期记忆（关键事实）
  Dream.run() → memory/MEMORY.md → 下次对话自动加载
```

#### MemoryStore — 纯文件 I/O 层

```python
class MemoryStore:
    memory_file = workspace / "memory" / "MEMORY.md"   # 长期记忆
    history_file = workspace / "memory" / "history.jsonl"  # 摘要历史
    soul_file = workspace / "SOUL.md"                   # Agent 人格
    user_file = workspace / "USER.md"                   # 用户信息
```

#### Consolidator — 实时摘要引擎

**触发条件**（[memory.py:551]()）：
```python
budget = context_window_tokens - max_completion_tokens - safety_buffer
if estimated < budget:
    return  # 不触发
# estimated >= budget → 开始压缩
```
仅在上下文 token 超出安全预算时才触发（约 93% 窗口使用率）。

**压缩流程：**
1. 选取用户轮次边界（不打断 tool call 的连续性）
2. 将旧消息通过 LLM 压缩为摘要
3. 摘要写入 `history.jsonl`
4. 更新 `session.last_consolidated` 游标
5. 摘要存入 `session.metadata["_last_summary"]`

#### Dream — 长期记忆提取

**两阶段执行**（[memory.py:773]()）：

**Phase 1（纯 LLM 分析）：**
```
输入: history.jsonl 未处理条目 + 当前 MEMORY.md（带行年龄标注）
输出: [FILE] / [FILE-REMOVE] / [SKILL] / [SKIP] 指令
```

**Phase 2（AgentRunner 执行）：**
- 使用 read_file / edit_file / write_file 工具精确修改 MEMORY.md
- 修改粒度是行级别的（不重写整个文件）
- 完成后 git 自动提交

**行年龄标注**（[memory.py:727]()）：
- MEMORY.md 每行超过 14 天未修改会标记 `← 30d`
- 提示 LLM 该行可能过时需要审核
- SOUL.md / USER.md 不标注（视为永久性内容）

#### 当前记忆管道的延迟问题

```
Consolidator 触发门槛 = context_window - completion - buffer
                     ≈ 65536 - 8192 - 2000 ≈ 55K tokens

短对话可能永远达不到这个阈值
→ history.jsonl 一直为空
→ Dream 无事可做
→ MEMORY.md 一直为模板
```

## 五、面试核心设计点

### 5.1 Agent 循环（ReAct 模式）

```
Think → Act → Observe → Think → ... → Final Answer
```

对应 [AgentRunner.run()](agent/runner.py#L227) 的 `for iteration` 循环。

**关键设计：**
- LLM 响应中 `tool_calls` 非空 → 执行工具 → 结果注入 → 继续循环
- LLM 响应中 `tool_calls` 为空 → 最终回复 → 停止
- 最大迭代次数限制（默认 200）防止无限循环
- 空回复自动重试 + length 截断自动恢复

### 5.2 上下文窗口管理

多层策略应对有限上下文：

| 策略 | 触发条件 | 机制 |
|------|----------|------|
| Tool Result 裁剪 | 单结果 > 16000 字符 | `_apply_tool_result_budget()` |
| Micro Compact | 重复工具调用 | 折叠为占位符 |
| History 裁剪 | 总 token > context_window | `_snip_history()` 头裁 |
| Consolidator | token > 93% 预算 | LLM 摘要压缩 |

### 5.3 记忆分层设计

| 层级 | 存储 | 生命周期 | 访问方式 |
|------|------|----------|----------|
| 短期 | Session JSONL | 跨重启持久 | 完整注入上下文 |
| 中期 | history.jsonl | 追加不删 | "# Recent History" 注入 |
| 长期 | MEMORY.md | 永久（Git 版本化） | 每次对话加载 |

### 5.5 多 Provider 抽象

统一的 `LLMProvider` 接口支持：
- OpenAI 兼容 API（DeepSeek / OpenRouter / Groq / ...）
- Anthropic Claude API（含 extended thinking）
- Azure OpenAI
- GitHub Copilot / OpenAI Codex CLI

重试策略：指数退避 + 瞬态错误识别 + 持久错误上限。

### 5.6 并发与状态管理

- **全局并发限制**：`Semaphore(3)` 防止 LLM API 过载
- **Session 串行**：同一 session 的消息排队处理（pending_queue）
- **Mid-turn Injection**：用户在同 session 处理过程中发送新消息时注入到进行中的 LLM 对话
- **任务可取消**：`/stop` 命令响应
