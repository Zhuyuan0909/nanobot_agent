# CloudSync 多平台智能客服 Bot — 整体设计分析

## 项目概览

基于 **nanobot** 框架二次开发的**多平台智能客服系统**，面向中国市场的企业即时通讯平台（飞书 + 钉钉），为虚构的 SaaS 公司 CloudSync（数据集成平台）提供自动化产品技术支持。

---

## 核心架构分层

| 层级 | 组件 | 说明 |
|---|---|---|
| **身份层** | `AGENTS.md` + `SOUL.md` + `USER.md` | 定义 Agent 人设、服务原则、用户画像 |
| **技能层** | `skills/faq/` `skills/memory/` `skills/ticket/` | 3 个 SKILL.md 通过 nanobot 的 skill 机制注入 LLM 上下文 |
| **工具层** | `mcp_servers/ticket_server.py` | 独立的 MCP JSON-RPC 2.0 服务，通过 stdio 与 nanobot 通信 |
| **知识层** | `knowledge_base/*.md` | 4 个 Markdown 文件充当 FAQ 知识库 |
| **数据层** | `data/tickets.db` (SQLite) + `memory/MEMORY.md` | 工单持久化 + 用户记忆 |
| **通道层** | `config.json` 配置飞书/钉钉 | 双通道接入，统一会话 |
| **调度层** | `cron/jobs.json` + `HEARTBEAT.md` | nanobot 内置 cron 机制 |

## 关键设计亮点

### 1. 5 步工作流闭环

读取记忆 → 分类问题 → FAQ/工单/升级路由 → 回复用户 → 更新记忆

### 2. 工单全生命周期管理

- 创建/查询/更新/解决
- 升级 SLA 分级：紧急 30min / 高 4h / 中 24h / 低 72h
- 满意度回访：1-5 分，低分自动重开工单

### 3. 跨平台统一会话

`unifiedSession: true`，同一用户在飞书和钉钉间切换不丢失上下文

### 4. MCP 工单服务器

用 MCP 协议将工单 CRUD 从 Agent 主循环中解耦，可独立扩展。实现 5 个工具：
- `create_ticket` — 创建工单
- `get_ticket` — 查询工单
- `list_user_tickets` — 列出用户工单
- `update_ticket` — 更新工单状态/备注
- `submit_feedback` — 提交满意度评分

### 5. 文件即知识库

FAQ 用 Markdown 文件存储，Agent 通过 grep/read_file 实时检索，无需向量数据库

### 6. Skill 驱动的 Agent 行为

3 个 always-on skill 全文注入 system prompt，教 LLM 如何使用工具、搜索知识库、管理工单、记录记忆

## 技术栈

| 层级 | 技术 |
|---|---|
| **Agent 框架** | nanobot v0.1.5+ (Python, HKUDS 开源) |
| **LLM 提供商** | DeepSeek (`deepseek-chat`) via `api.deepseek.com` |
| **通信通道** | 飞书 SDK (WebSocket 长连接), 钉钉 SDK |
| **工具协议** | MCP (Model Context Protocol) — JSON-RPC 2.0 over stdio |
| **数据库** | SQLite (`workspace/data/tickets.db`) |
| **记忆存储** | 文件 Markdown (`memory/MEMORY.md`) + JSONL 会话日志 |
| **部署** | Docker (`python:3.12-slim-bookworm`) + docker-compose |
| **语言/框架** | Python 3.11+, nanobot-ai PyPI 包 |
| **会话管理** | JSONL 文件 (跨通道统一会话) |
| **调度** | nanobot 内置 cron + Dream 记忆维护 |

## 面试价值分析

**比较有说服力的点：**
- 不是 Demo 级的单文件脚本，有清晰的架构分层
- MCP 协议集成是当前 Agent 领域的热门话题
- 多平台 + 统一会话在实际业务中确实有价值
- 工单 SLA 分级 + 满意度回访体现了产品思维

**可能需要补强的方面：**
- 知识库检索目前是 grep 纯文本匹配，缺少语义搜索
- 没有做对话质量监控/统计
- 缺少自动化测试
